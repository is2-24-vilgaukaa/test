1.1 Цель

Цель написания отчета состоит в предоставлении полной и объективной информации о текущем состоянии качества проекта Paint.

Этот документ используется для записи процессов тестирования, суммирует условия испытаний каждого раунда, анализирует тестовые данные, индукцию вопросов и устаревших рисков, подверженных воздействию во время тестирования и дает соответствующие рекомендации по тестированию для последующей деятельности.

2.4 Прецедент

Прецедент отражает функциональные требования к системе, описывает ее границы, интерфейс, и выступает основой для тестирования заказчиком.

В ходе выполнения задачи были использованы следующие варианты использования прецедентов:

·     основной (выполнение функций);

·     вспомогательный (настройка системы);

·     дополнительный (для удобства пользователя).

2.5 Тестовая стратегия

Тестовая стратегия для тестирования приложения Paint:

1. Тип теста: определяется как интегрированное тестирование и тестирование системы.

2. Интегрированная тестовая фаза была интегрирована в основном для анализа, подтверждения и нахождение несоответствия спроса в качестве основной цели.

3. Фаза тестирования системы разделена на три раунда. Основная стратегия заключается в следующем:

Первый раунд - это тест на покрытие, а тестовый диапазон охватывает все диапазоны, описанные выше, обращающие внимание на все уровни ошибок;

Второй раунд представляет собой функциональный тест, который ослабляет все отремонтированные ошибки;

Третий раунд функционально проверен, совместимый с модулями результатов. Дымовое тестирование выполняется до начала каждого раунда теста. Тест по совместимости и тест интерфейса выполняется только в первом раунде.

4. Оценка дефекта: после каждого раунда теста оно организует разработку инженеров, тестовых инженеров, инженеров продуктов и других дефектов продукта совместной оценки. Содержание оценки включает в себя дефектные решения, включает в себя изменения в востребовании, следующий раунд времени начала и может ли это закончить тест.
